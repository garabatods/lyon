# Chameleon Puzzle Docs

Read `00_CURRENT_PROJECT_STATE.md` first. It describes the live game after the latest gameplay and 6x8 board refactors.

The numbered docs are historical source material from the original prototype prompt. They still help explain where the project came from, but they contain outdated references to:

- 5x7 board sizing,
- gesture-first controls,
- color mixing,
- green as an active color,
- passive match-3 clears.

Recommended read order for future work:

1. `00_CURRENT_PROJECT_STATE.md`
2. Root `README.md`
3. `06_POWER_UPS_SPEC.md` when working on the bottom HUD, power-up slots, rewards, or tutorial progression
4. Current code and tests
5. Historical numbered docs only when useful for asset/background context

When docs conflict, prefer `00_CURRENT_PROJECT_STATE.md`, `06_POWER_UPS_SPEC.md` for power-up design, root `README.md`, and the current tests.
